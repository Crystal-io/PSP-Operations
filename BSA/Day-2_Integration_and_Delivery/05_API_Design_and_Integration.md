# 05. API Design и интеграции

## Цель модуля
Научиться описывать интеграции так, как ожидают от системного аналитика.

---

## 1. API как контракт
API — это формальное соглашение между системами.

Аналитик описывает:
- endpoint
- метод
- формат данных
- ошибки
- ограничения

---

## 2. Swagger / OpenAPI
Используется для:
- документирования API
- согласования контрактов
- тестирования

---

## 3. Типовая структура интеграции

System A → REST API → System B

Важно учитывать:
- авторизацию
- таймауты
- ретраи
- ошибки

---

## 4. Ошибки интеграций
- неверные данные
- сетевые сбои
- бизнес-ошибки

---

## Результат
Ты умеешь описывать интеграции понятным языком.
